## Part 1 — Unblocking `ingest_deputies.py`

### Problem 1: `python` command not found

The system uses Python 3.14. The `python` binary is not in PATH — only `python3`. The fix was to always call `venv/bin/python3` explicitly rather than relying on shell activation.

### Problem 2: `psycopg2-binary==2.9.9` fails to build on Python 3.14

The `requirements.txt` pins `psycopg2-binary==2.9.9`. PyPI does not ship a pre-built wheel for that version on Python 3.14 / macOS ARM64. Attempting to install it triggers a source build which fails immediately with:

```
Error: pg_config executable not found.
```

**Fix:** Install `psycopg2-binary>=2.9.10` first (version 2.9.11 has an ARM64 wheel for cp314), then install all other dependencies individually, skipping the pinned psycopg2 entry:

```bash
venv/bin/pip install "psycopg2-binary>=2.9.10"
venv/bin/pip install fastapi==0.111.0 "uvicorn[standard]==0.29.0" sqlalchemy==2.0.30 \
    requests==2.32.2 python-dotenv==1.0.1 httpx==0.27.0 pandas==2.2.2
```

**Action item for the repo:** Update `requirements.txt` line 3 from `psycopg2-binary==2.9.9` to `psycopg2-binary>=2.9.10`.

### Problem 3: The AN REST API does not exist

The original `ingest_deputies.py` called:

```
GET https://data.assemblee-nationale.fr/api/v2/acteurs/deputes?page=1&limit=100
```

This returns **HTTP 404**. After inspecting `https://data.assemblee-nationale.fr/acteurs/deputes-en-exercice`, it became clear that the Assemblée Nationale open-data portal exposes **no REST API at all**. All data is served as static downloadable ZIP archives.

The correct deputy data URL is:

```
https://data.assemblee-nationale.fr/static/openData/repository/17/amo/deputes_actifs_mandats_actifs_organes/AMO10_deputes_actifs_mandats_actifs_organes.json.zip
```

**The rewrite:** `fetch_all_deputies()` was replaced entirely. Instead of paginating a REST endpoint, the new version:
1. Downloads the ZIP into memory (`io.BytesIO`)
2. Opens it with `zipfile.ZipFile`
3. Lists all files matching `json/acteur/*.json` (577 files)
4. Parses each file as `{"acteur": {...}}` and extracts the inner `acteur` dict
5. Feeds the list to the existing `parse_deputy()` function unchanged

The `parse_deputy()` parser was already written to handle the correct data shape (it was just never reachable due to the wrong data source).

### Result

```
2026-04-01 [INFO] ZIP contains 577 actor files.
2026-04-01 [INFO] Total deputies fetched: 577
2026-04-01 [INFO] Parsed 577 valid records (skipped 0 unparseable).
2026-04-01 [INFO] Upsert complete — 577 deputies written.
```

- A single `votant` in a block comes as a `dict`, not a `list` — must normalise to list
- `groupe` can be a single `dict` (if only one groupe voted) or a `list` — must normalise

## Part 5 — `scripts/ingest_positions.py` (new file)

This was the most complex script. It reads the same `Scrutins.json.zip` and extracts individual deputy voting positions.

### Data path

```
scrutin
  └── ventilationVotes
        └── organe
              └── groupes
                    └── groupe[]          ← list of political groups
                          └── vote
                                └── decompteNominatif
                                      ├── pours        → position: "pour"
                                      ├── contres      → position: "contre"
                                      ├── abstentions  → position: "abstention"
                                      └── nonVotants   → position: "nonVotant"
```

Each block (`pours`, `contres`, etc.) is either `null` or `{"votant": [...]}`. Each `votant` has `acteurRef` which maps to `deputies.deputy_id`.

### Key design decisions

**FK safety:** Before processing, load all `vote_id` and `deputy_id` values from the DB into sets. Skip any position where either FK is missing. This avoids FK violation errors and is faster than catching exceptions per row.

**Batched upsert:** Use `psycopg2.extras.execute_batch()` with `page_size=500` instead of row-by-row inserts. Flush every 2,000 positions. This gave ~10x throughput improvement over individual `cur.execute()` calls.

**Deduplication:** Track a `seen` set of `deputy_id` per scrutin. If a deputy appears in multiple groups (which shouldn't happen but can in edge cases), take only the first occurrence.

**Skipped positions:** 31,528 positions were skipped. These are deputies from previous legislatures (legislature 16 and earlier) who appear in some scrutin files but are not in our `deputies` table (which only contains currently active deputies from legislature 17).

### Result

```
2026-04-01 [INFO] Known votes: 5922  Known deputies: 577
2026-04-01 [INFO] ZIP contains 5922 scrutin files.
2026-04-01 [INFO] Upsert complete — 948,217 positions written, 31,528 skipped.
```

## Part 7 — `GET /deputies/{id}/scorecard`

Added to `api/routers/deputies.py`. Computes real stats from `vote_positions`:

```sql
SELECT
    COUNT(*)                                          AS total_votes,
    COUNT(*) FILTER (WHERE position != 'nonVotant')  AS present_votes,
    COUNT(*) FILTER (WHERE position = 'pour')        AS votes_for,
    COUNT(*) FILTER (WHERE position = 'contre')      AS votes_against,
    COUNT(*) FILTER (WHERE position = 'abstention')  AS abstentions
FROM vote_positions
WHERE deputy_id = %s
```

Computed fields:
- `presence_rate` = `present_votes / total_votes` (rounded to 4 decimal places)
- `votes_for_pct` = `votes_for / present_votes`
- `abstention_pct` = `abstentions / present_votes`

## Known Issues / Next Steps (Phase 2 candidates)

1. **`party` is null everywhere.** The active deputies ZIP does not include group names — only `organeRef` IDs. Resolving those to human-readable names (RN, NFP, EPR, etc.) requires downloading and joining the organes export from `/acteurs/organes`.

2. **`psycopg2-binary==2.9.9` in requirements.txt** will fail on Python 3.14. Should be updated to `>=2.9.10`.

3. **No connection pooling.** Each API request opens and closes a new `psycopg2` connection. For production this should be replaced with SQLAlchemy connection pooling or `asyncpg` + `databases`.

4. **Scorecard missing `most_common_position`.** The schema defines it but it was not added in Phase 1. Easy to add: `SELECT position FROM vote_positions WHERE deputy_id = %s GROUP BY position ORDER BY COUNT(*) DESC LIMIT 1`.

5. **No search on votes.** `GET /votes/` only supports `result` filter. A `search` param on `vote_title` (ILIKE) would be useful.

6. **Legislature hardcoded.** The ZIP paths contain `/17/` (17th legislature). This will break when the 18th begins. Should be made configurable via env var.

7. **`voted_at` stored as DATE cast to TIMESTAMPTZ.** The AN data only provides a date string (`2025-06-24`), not a timestamp. PostgreSQL stores it as midnight UTC. This is correct but should be documented.

8. **No incremental ingestion.** Every run downloads the full ZIP and re-upserts everything. For votes/positions this takes ~2 minutes. An incremental mode (only fetch new scrutins since last run) would be faster once the initial load is done.
