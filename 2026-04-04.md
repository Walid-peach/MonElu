## 1 — `railway.json`

Created at project root. Tells Railway how to build and run the app:

```json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": { "builder": "NIXPACKS" },
  "deploy": {
    "startCommand": "uvicorn api.main:app --host 0.0.0.0 --port $PORT",
    "healthcheckPath": "/health",
    "restartPolicyType": "ON_FAILURE"
  }
}
```

Key points:
- `NIXPACKS` auto-detects Python and installs from `requirements.txt` — no `Dockerfile` needed
- `$PORT` is injected by Railway at runtime — never hardcode a port
- `healthcheckPath` means Railway will poll `/health` after each deploy and only mark it live once it returns 200

## 4 — `api/main.py` — enriched `/health`

Before:
```python
@app.get("/health")
def health() -> dict:
    return {"status": "ok"}
```

After: opens a DB connection, queries all three tables, and returns live counts:

```python
@app.get("/health")
def health() -> dict:
    try:
        conn = psycopg2.connect(os.getenv("DATABASE_URL"))
        with conn.cursor() as cur:
            cur.execute("SELECT COUNT(*) FROM deputies")
            deputies = cur.fetchone()[0]
            cur.execute("SELECT COUNT(*) FROM votes")
            votes = cur.fetchone()[0]
            cur.execute("SELECT COUNT(*) FROM vote_positions")
            positions = cur.fetchone()[0]
        conn.close()
        return {"status": "ok", "deputies": deputies, "votes": votes, "positions": positions}
    except Exception as exc:
        return {"status": "degraded", "error": str(exc)}
```

The `degraded` branch is important — Railway's healthcheck expects a 200 response. Returning `{"status": "degraded"}` with HTTP 200 lets the app stay up while still surfacing the DB error. A hard exception would return 500 and cause Railway to restart the container in a loop.

Verified response:
```json
{"status": "ok", "deputies": 578, "votes": 6022, "positions": 960298}
```

Added to `api/routers/votes.py`. Returns the 10 most recent votes, no pagination needed.

## 6 — `scripts/run_ingestion_prod.py`

Designed to be run as a one-off Railway job after the first deploy to populate the DB.

### First attempt — `importlib.import_module`

The script originally tried to import the other scripts as Python modules:

```python
mod = importlib.import_module("scripts.ingest_deputies")
mod.main()
```

This failed with `ModuleNotFoundError: No module named 'scripts'` because `scripts/` has no `__init__.py` and is not a package on `sys.path`.

### Fix — `subprocess.run`

Replaced with subprocess execution. Each script runs in its own process, which is actually better:
- Clean environment per step
- Output streams directly to terminal in real time
- A crash in one step raises `RuntimeError` and halts the chain rather than corrupting the parent process state

```python
subprocess.run([sys.executable, script_path], cwd=PROJECT_ROOT, env={**os.environ})
```

`sys.executable` ensures the same Python interpreter (and therefore the same venv) is used regardless of how the runner is invoked.

### Output format

```
╔══════════════════════════════════════╗
║         INGESTION COMPLETE           ║
╠══════════════════════════════════════╣
║  Deputies  :    578   (  2.6s)      ║
║  Votes     :   6022   (  4.6s)      ║
║  Positions : 960298   ( 53.6s)      ║
╠══════════════════════════════════════╣
║  Total time: 60.8s                   ║
╚══════════════════════════════════════╝
```

After each step, counts are read from the DB (not from script return values) so the summary always reflects what is actually in the database.
